# 智慧大棚温湿度预测系统 · 教学使用手册

> **仅用于教学与学习交流**  

---

## 一、系统简介

智慧大棚温湿度预测系统是一个基于 Flask 开发的 Web 应用，用于展示物联网传感器数据的上传、清洗、聚合以及基于机器学习/时间序列模型的温湿度未来趋势预测。系统模拟了真实农业物联网平台的核心流程，包括：

- 用户注册/登录/权限管理（管理员、操作员、普通用户）
- 原始 TXT 传感器数据自动清洗与格式转换
- 多传感器数据聚合（整体大棚）
- 基于 Prophet（年周期）和线性回归（滑动窗口）的温湿度预测
- 预测结果图表生成与 CSV 导出
- 站内消息系统
- 管理员后台（用户管理、数据文件管理、预测历史管理、系统参数配置、登录日志）

**适用场景**：智慧农业、物联网数据处理、Web 开发、数据分析等课程的教学演示与实验。

## 二、快速开始

### 2.1 环境要求
- Python 3.10+
- 建议使用虚拟环境（venv）

### 2.2 安装依赖
```bash
# 创建并激活虚拟环境（Windows PowerShell）
python -m venv .venv
.\.venv\Scripts\Activate

# 安装所需库（注意 numpy 版本）
pip install -r requirements.txt
```

### 2.3 启动系统
```bash
python app.py
```
启动成功后，浏览器访问：**http://127.0.0.1:5000**

### 2.4 默认管理员账号
- 用户名：`admin`
- 密码：`admin123`

> 首次启动会自动创建数据库并初始化管理员账户。

## 三、系统功能与操作步骤

### 3.1 用户注册与登录
1. 访问首页，点击右上角 **“注册”** 按钮。
2. 填写用户名（3-20位字母、数字或下划线）和密码（至少6位）。
3. 注册成功后自动跳转登录页，输入账号密码登录。
4. 登录后进入系统首页（欢迎页）。

### 3.2 首页功能
- **欢迎卡片**：显示当前登录用户及角色。
- **快速开始**：提供四步引导（上传数据 → 选择模型 → 生成预测 → 历史记录）。
- **管理员资料**：展示管理员/操作员上传的公告或资料附件（仅管理员/操作员可上传，点击右上角 ⊕ 按钮）。

### 3.3 预测中心（核心功能）
1. 点击导航栏 **“预测中心”**。
2. **上传数据文件**：
   - 支持 `.txt` 格式的温湿度数据（格式：序号、温度、湿度、日期，制表符分隔）。
   - 可多选文件批量上传。
   - 拖拽文件至上传区域或点击“选择文件”按钮。
3. **选择预测目标**：温度(℃) 或 湿度(%)。
4. **选择模型**：
   - 自动推荐（根据系统内已有数据跨度自动选择）
   - Prophet（适合 ≥365 天数据，年周期预测）
   - 线性回归（适合 <365 天数据，短期滑动窗口预测）
5. 点击 **“开始批量预测”**，系统将：
   - 自动清洗上传的 TXT 文件（剔除异常值、连续零值等），转换为 CSV 存入 `温湿度数据csv/` 目录。
   - 对每个文件独立进行预测，并同时生成一份基于**所有历史传感器数据**的“整体大棚预测”。
6. 预测完成后跳转至 **批量预测结果** 页面，可查看：
   - 整体大棚预测结果（图表 + CSV 下载）
   - 每个上传文件的独立预测结果（图表 + CSV 下载）

### 3.4 个人中心
- 查看/修改密码。
- 查看个人历史预测记录（支持下载 CSV、单条删除、清空全部记录）。

### 3.5 消息系统
- 左侧选择收件人并发送消息。
- 右侧收件箱显示收到的消息（未读消息高亮），发件箱显示已发送消息。
- 点击在线用户列表可快速填充收件人。

### 3.6 管理员后台（管理员/操作员可见）
导航栏下拉菜单 **“管理”** 中包含以下子模块：

| 模块 | 功能 | 权限 |
| :--- | :--- | :--- |
| **数据文件** | 查看 `温湿度数据csv/` 目录下所有 CSV 文件，支持批量删除、手动触发整体大棚数据聚合、检查文件清洗效果 | 管理员、操作员 |
| **预测历史** | 查看所有用户的预测记录，支持按用户筛选、批量删除、清理无效记录（文件丢失的记录） | 管理员、操作员 |
| **用户管理** | 查看/修改用户角色（管理员/操作员/查看者/用户）、启用/禁用账号、删除用户、批量导入用户（CSV） | 仅管理员 |
| **系统设置** | 配置预测参数：默认模型、Prophet 预测天数、线性回归预测天数、自动推荐阈值、Prophet 变点敏感度 | 仅管理员 |
| **登录日志** | 查看所有用户的登录成功/失败记录（IP、时间） | 仅管理员 |

## 四、数据文件格式说明

### 上传的 TXT 文件格式要求
- 文件编码：UTF-8、GBK 等均可（系统自动识别）。
- 必须包含表头行，且包含关键词：`序号`、`温度`、`湿度`、`日期`。
- 数据行使用**制表符（Tab）**或**空格**分隔。
- 示例：
```
序号	温度(℃)	湿度(%)	日期
1	23.5	65.2	2025-01-01 00:00:00
2	23.4	65.5	2025-01-01 01:00:00
```

### 数据清洗规则（自动执行）
- 删除温度 > 50℃ 的异常值。
- 删除湿度 < 1% 的异常值。
- 删除温度=0 且湿度=0 的行。
- 删除连续三行温度均为 0 或连续三行湿度均为 0 的行。
- 过滤后有效记录数不足则报错。

## 五、预测模型说明

| 模型 | 适用数据跨度 | 预测时长 | 核心原理 |
| :--- | :--- | :--- | :--- |
| **Prophet** | ≥365 天 | 默认 30 天（可配置） | Facebook 开源时序预测，自动处理年/周/日周期 |
| **线性回归（滑动窗口）** | <365 天 | 默认 7 天（可配置） | 使用过去 24 小时数据预测下一小时，迭代预测 |

管理员可在“系统设置”中调整预测天数及自动推荐阈值。

## 七、注意事项

1. **首次运行**：若提示数据库表缺少字段（如 `user.status`），请删除 `instance/users.db` 文件后重新启动。
2. **NumPy 版本**：建议使用 `numpy==1.26.4`，避免与 pandas、scikit-learn 等库不兼容。
3. **中文显示**：预测图表已配置中文字体，若服务器环境无中文字体，图表中文可能显示为方块，需安装相应字体（如 SimHei）。
4. **文件上传大小**：Flask 开发服务器默认无限制，但生产环境建议配置 `MAX_CONTENT_LENGTH`。
5. **教学用途**：本系统仅为演示教学，不保证预测结果在实际农业生产中的准确性，请勿用于真实农事决策。
6. **安全警告**：开发服务器 `debug=True` 仅限本地学习，部署到公网请更换 `SECRET_KEY` 并关闭 Debug 模式。

## 八、常见问题排查

| 问题 | 解决方法 |
| :--- | :--- |
| 启动时报 `no such column: user.status` | 删除 `instance/users.db` 后重启 |
| 预测图表中文乱码 | 确保系统安装中文字体，或修改 `app.py` 中的 `plt.rcParams['font.sans-serif']` |
| 上传文件后预测失败 | 检查 TXT 文件是否符合格式要求，查看终端错误日志 |
| 无法访问管理后台 | 确认当前登录用户角色为 `admin` 或 `operator` |
| 忘记管理员密码 | 删除数据库后重启，系统将重新生成默认账号 `admin/admin123` |
